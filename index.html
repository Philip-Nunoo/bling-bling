Model | View | Controller

Ruby on Rails:

The RESTAURANT Analogy

At the core of Rails is the Model, View, Controller architecture (MVC).
MVC benefits include:

  • Isolation of business logic from the user interface
  • Ease of keeping code DRY
  • Making it clear where different types of code belong for easier maintenance.

Model

The Chef


The Chef receives orders from the waiter and creates the dish using various ingredients in the kitchen.

The model represents the information of the application and the rules to manipulate that information.










View

The Table

The glorious tablespread of the restaurant patron's dinner. Or, the user interface of the application.

The views are often HTML files with embedded Ruby code that perform tasks related solely to the presentation of the data.









Controller

The Waiter

The waiter takes the table's (user's) order, along with any special vegeterian or allergy requests, and brings it to the kitchen.

The controllers are responsible for processing the incoming requests from the web browser, interrogating the models for data, and passing that data on to the views for presentation.
